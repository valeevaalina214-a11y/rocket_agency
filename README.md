# Rocket Agency

Лендинг креативного блогерского агентства Rocket Agency. Проект содержит готовый адаптивный интерфейс, карточки блогеров, услуги, экосистему Rocket Group, контакты, два фоновых видео и анимации появления при прокрутке.

## Что входит в проект

- адаптивная главная страница для десктопа, планшета и телефона;
- сортируемый список блогеров с раскрытием остальных карточек;
- кликабельные карточки экосистемы Rocket Group;
- изображения блогеров и логотипы в `public/`;
- два локальных видео персонажей в `public/videos/`;
- единый лавандовый фон, звёзды, свечение карточек и кастомный курсор;
- доступные состояния клавиатуры и поддержка `prefers-reduced-motion`.

## Технологии

- React 19 и Next.js 16;
- TypeScript;
- Tailwind CSS 4;
- Framer Motion;
- Vinext/Vite для сборки под Cloudflare Workers.

## Требования

- Node.js `22.13.0` или новее;
- npm;
- Linux, macOS или WSL. Вспомогательные скрипты сборки используют Bash.

## Локальный запуск

```bash
npm ci
npm run dev
```

После запуска откройте адрес, который будет показан в терминале.

## Проверка перед публикацией

```bash
npm run lint
npm run typecheck
npm run build
```

## Основные папки

```text
app/          страница, layout и глобальные стили
components/   секции и интерактивные компоненты
data/         тексты, ссылки и данные карточек
lib/          общие настройки анимаций
public/       фотографии, логотипы, видео и favicon
scripts/      проверенные команды установки и сборки
tests/        автоматические проверки
worker/       точка входа Cloudflare Worker
```

## Редактирование контента

- блогеры: `data/bloggers.ts`;
- услуги: `data/services.ts`;
- экосистема и ссылки: `data/rocketGroup.ts`;
- контакты: `data/contacts.ts`;
- общие данные сайта: `data/site.ts`;
- визуальные стили: `app/globals.css` и компоненты в `components/`.

## Перенос в GitHub

Пошаговая инструкция находится в [GITHUB_TRANSFER.md](GITHUB_TRANSFER.md). Репозиторий можно хранить на GitHub бесплатно. Обычный GitHub Pages не запускает серверную Vinext/Worker-сборку; для действующего сайта используйте текущий хостинг Sites либо подключите репозиторий к платформе, поддерживающей Cloudflare Workers.

## Важное об ассетах

Все используемые сайтом файлы находятся внутри `public/`, поэтому после клонирования изображения, логотипы и видео не потеряются. Перед публичной публикацией убедитесь, что у вас есть права на размещение фотографий, логотипов и видеоматериалов.
